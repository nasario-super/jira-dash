# Performance Testing Guide

This guide provides an overview of performance and load testing scenarios for the Jira Dashboard. The focus is on optimizing processing times and ensuring efficient data handling across multiple projects.

## Introduction

Performance testing is crucial for ensuring that applications can handle expected loads efficiently. This guide outlines the steps and improvements made to optimize the Jira Dashboard, reducing processing times and enhancing data handling capabilities.

## Performance Optimization

### Data Fetching and Processing

The new process for fetching and merging data from multiple projects has been significantly optimized. Previously, processing times exceeded 100 seconds, but through parallel data fetching, this has been reduced to just 5-10 seconds, representing a 10-20x performance improvement.

- **Before Optimization**: 100+ seconds
- **After Optimization**: 5-10 seconds
- **Improvement**: 10-20x faster

### Testing Scenarios

#### Test 1: Performance with Two Projects

To test the performance improvements, follow these steps:

1. Open DevTools (F12) and navigate to the Console.
2. Log in to the application.
3. Select the projects INFOSECC and SEGP.
4. Observe the processing time:
   - The process should complete in 5-15 seconds.
   - The console should display "SUCCESS".
   - If the process takes more than 30 seconds, further investigation is needed.
