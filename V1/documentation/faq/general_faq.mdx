# General FAQ for Jira Dashboard

Welcome to the FAQ page for the Jira Dashboard. This document aims to address the most common questions and provide guidance on using the Jira Dashboard effectively. The Jira Dashboard is a web-based tool that centralizes Jira data, offering advanced analytics and reporting capabilities.

## What is the Jira Dashboard?

The **Jira Dashboard** is a modern web tool designed to provide a centralized view of Jira data. It offers several key features:

- Centralized data visualization from Jira
- Advanced productivity analytics
- Executive reports
- Quality metrics
- AI insights
- Forecasts and anomaly detection

## Getting Started

### Accessing the Application

To access the Jira Dashboard, visit the following URL:

```
http://3.83.28.223:3000
```

You will be presented with the login screen.

### Login Requirements

To log in, you will need:

- **Email**: Your registered email address
- **API Token**: This can be generated at [Atlassian API Tokens](https://id.atlassian.com/manage-profile/security/api-tokens)

#### How to Generate an API Token

1. Visit [Atlassian API Tokens](https://id.atlassian.com/manage-profile/security/api-tokens)
2. Click on "Create API token"
3. Enter a name for the token (e.g., "Jira Dashboard")
4. Copy the generated token
5. Paste it on the login screen of the Dashboard

### Project Selection

After logging in, you will see the project selection screen.

#### Options Available:

1. Load Projects
2. Search by name
3. Select multiple projects
4. Confirm selection

**Tip**: Select 1-3 projects for optimal performance. You can select more later if needed.

## Common Issues and Solutions

### Issue Status Discrepancies

**Why?**
- Real-time updates
- Multiple users making changes
- Sync with Jira

**Solution**:
- Click **Update** on the Dashboard
- Or refresh the page (F5)

### Mismatched Issue Numbers

**Why?**
- Active filters
- Sub-item issues not included
- Ongoing synchronization

**Check**:
- Review active filters
- Look for "Filtros Aplicados" on the card

### Missing User

**Why?**
- User has no assigned issues
- Issues reassigned
- User is out of the project

**Action**: Reassign issues if necessary

## Technical Setup

### Environment Configuration

Edit the `.env` file with your Jira credentials:

```env
VITE_JIRA_DOMAIN=sua-empresa.atlassian.net
VITE_JIRA_EMAIL=seu-email@empresa.com
VITE_JIRA_API_TOKEN=seu-token-aqui
```

### Development Commands

```bash
npm run dev # Start development server
npm run build # Production build
npm run preview # Preview the build
npm run lint # Check for linting issues
```
