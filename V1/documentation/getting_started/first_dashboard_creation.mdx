# Creating Your First Jira Dashboard

Welcome to the Jira Dashboard tutorial. This guide will walk you through the process of creating your first dashboard, understanding its interface, and utilizing its features to enhance your productivity with Jira.

## Introduction to Jira Dashboard

The **Jira Dashboard** is a modern web tool designed to centralize Jira data, offering advanced analytics and executive reports. It provides users with a practical and intuitive way to access projects and visualize productivity metrics.

## Getting Started

### Accessing the Application

To begin, access the Jira Dashboard application through the following URL:

```
http://3.83.28.223:3000
```

Upon accessing, you will be greeted with the login screen.

### Logging In

#### Required Credentials:
- **Email**: your-email@example.com
- **API Token**: Generate at [Atlassian API Tokens](https://id.atlassian.com/manage-profile/security/api-tokens)

**How to Generate an API Token:**
1. Visit [Atlassian API Tokens](https://id.atlassian.com/manage-profile/security/api-tokens)
2. Click on "Create API token"
3. Name your token (e.g., "Jira Dashboard")
4. Copy the generated token
5. Paste it on the dashboard login screen

### Project Selection

After logging in, you will see the project selection screen.

#### Options:
- Load Projects
- Search by name
- Select multiple projects
- Confirm selection

**Tip:** Select 1-3 projects for optimal performance. You can select more projects later if needed.

## Dashboard Overview

### Main Features

- **Centralized Data Visualization**: View all your Jira data in one place.
- **Advanced Analytics**: Gain insights into productivity and quality metrics.
- **Executive Reports**: Generate detailed reports for management.
- **AI Insights**: Access predictions and anomaly detection.

### Interface Components

#### Filter and Visualization Integration

The dashboard integrates filters and visualizations, processing data for various charts. It features a filter bar and other UI elements styled with Tailwind CSS.

#### Key Components

- **Header**: Displays the main navigation and user options.
- **Filter Bar**: Allows you to apply and manage filters on your data.
- **Charts**: Visualize data through various chart types such as burndown, status distribution, velocity, and more.

### Common Issues and Solutions

1. **Issue Status Discrepancy**
   - **Cause**: Real-time updates, multiple users, Jira synchronization.
   - **Solution**: Click "Update" on the Dashboard or refresh the page.

2. **Mismatch in Issue Count**
   - **Cause**: Active filters, sub-item issues not included, ongoing synchronization.
   - **Check**: Review active filters and the "Applied Filters" card.

3. **Missing User**
   - **Cause**: User has no assigned issues, issues reassigned, user not in the project.
   - **Action**: Reassign issues as necessary.

## Technical Setup

### Prerequisites

- **Node.js 18+**
- **npm or yarn**
- **Jira Cloud Account** with API permissions
- **API Token** from Jira

### Installation

1. **Clone the Repository**
   ```bash
git clone <repository-url>
cd jira-dashboard
   ```

2. **Install Dependencies**
   ```bash
npm install
```

# or
yarn install
   ```

3. **Configure Environment Variables**
   Copy the example file and edit with your Jira credentials:
   ```
cp env.example .env

### Deployment

The Jira Dashboard is ready for deployment on AWS EC2 with a simple three-step process:

1. **Clone the Repository on EC2**
   ```bash
cd /home/ssm-user/projects
git clone https://github.com/your-username/jira-dash.git
cd jira-dash
   ```

2. **Run the Deployment Script**
   ```bash
chmod +x install-and-deploy.sh
./install-and-deploy.sh
   ```

3. **Access the Application**
   ```
http://YOUR-EC2-IP:3000
   ```
