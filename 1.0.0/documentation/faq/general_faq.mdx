# General FAQ for Jira Dashboard

## Introduction

Welcome to the General FAQ for Jira Dashboard, a modern web tool designed to centralize Jira data, provide advanced analytics, and generate executive reports. This documentation aims to address the most common questions and issues users may encounter while using Jira Dashboard.

## What is Jira Dashboard?

Jira Dashboard is a web-based application that offers a centralized view of Jira data. It provides advanced productivity analytics, executive reports, quality metrics, AI insights, and anomaly predictions. This tool is designed to facilitate project selection and metric visualization in a practical and intuitive manner.

## Getting Started

### Accessing the Application

To access the Jira Dashboard, visit the following URL:

```
http://3.83.28.223:3000
```

Upon visiting, you will be greeted with the login screen.

### Login Requirements

To log in, you need the following credentials:
- **Email**: your-email@example.com
- **API Token**: Generate at [Atlassian API Tokens](https://id.atlassian.com/manage-profile/security/api-tokens)

#### Steps to Generate an API Token

1. Visit [Atlassian API Tokens](https://id.atlassian.com/manage-profile/security/api-tokens)
2. Click on "Create API token"
3. Enter a name (e.g., "Jira Dashboard")
4. Copy the generated token
5. Paste it on the login screen of the Dashboard

### Project Selection

After logging in, you will see the project selection screen. Here are your options:
- Load Projects
- Search by name
- Select multiple projects
- Confirm selection

**Tip**: For optimal performance, select 1-3 projects initially. You can select more later if needed.

## Common Issues and Solutions

### Issue Status Discrepancy

**Why?**
- Real-time updates
- Multiple users modifying
- Synchronization with Jira

**Solution**:
- Click **Update** on the Dashboard or refresh the page (F5).

### Mismatched Issue Numbers

**Why?**
- Applied filters
- Sub-item issues not included
- Synchronization in progress

**Check**:
- Review active filters
- Look for "Applied Filters" on the card

### Missing User

**Why?**
- User without assigned issues
- Issues reassigned
- User removed from the project

**Action**: Reassign issues if necessary.
